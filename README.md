## О проекте

**СтрахоКвест** - это геймифицированная обучающая платформа, которая помогает молодым людям принимать грамотные финансовые решения в ситуациях, связанных со страхованием. Пользователь проходит интерактивные сценарии (ДТП, затопление квартиры, кража гаджета и т.д.), делает выборы и видит реальные финансовые последствия каждого решения.

Ключевая идея: **«учись на чужих ошибках, не теряя реальные деньги»**.

---

## Демонстрация

Приложение состоит из трёх игровых режимов, доступных через нижнее меню:

| Режим | Иконка | Описание |
|-------|--------|----------|
| История | 📖 | AI-сгенерированный сценарий из 5 шагов с выбором решений |
| Квиз | 🧠 | Тест на знание страховых понятий |
| Свайп | 👆 | Карточная игра: стоит ли страховать этот предмет? |

---

## Бизнес-логика

### Основной игровой цикл

1. Пользователь вводит тему (например, «попал в аварию» или «украли телефон»)
2. Бэкенд обращается к **GigaChat API** и генерирует уникальный сценарий из 5 сцен
3. В каждой сцене предлагаются два варианта действий: **правильный** (с страховкой / грамотным поведением) и **неправильный**
4. Каждый выбор имеет финансовое последствие в рублях - деньги либо сохраняются, либо теряются
5. В конце показывается итоговая статистика: сколько удалось сберечь, сколько потеряно, и главные уроки

### Офлайн-режим (Демо)

Если GigaChat API недоступен (нет ключа, недостаточно баланса, сеть недоступна), сервер автоматически переключается на встроенную базу из **8+ готовых сценариев**. Подбор нужного сценария происходит по ключевым словам из запроса пользователя.

### Образовательный компонент

- После каждого выбора показывается объяснение (почему это хорошо или плохо)
- В каждой сцене есть **страховой факт** (factoid) с полезной информацией
- На экране результатов формируется персональный список «уроков»

## Функциональность

- **AI-генерация сценариев** через GigaChat (Сбер) с автоматическим фоллбэком
- **Финансовый трекер**: накопленные потери / сбережения в ₽ во время игры
- **Три режима обучения**: история, квиз, свайп
- **Подсказки-чипы** с готовыми темами на главном экране
- **Анимированный экран загрузки** с прогрессом генерации
- **Мобильная адаптация**: touch-friendly, bottom navigation, оптимизировано под смартфоны
- **Офлайн-режим**: работает без интернета и без API-ключа

## API

### `POST /api/generate-scenario`

Генерирует игровой сценарий по теме.

**Запрос:**
```json
{
  "topic": "попал в аварию на машине"
}
```

**Ответ (успех):**
```json
{
  "scenario": {
    "name": "ДТП на перекрёстке",
    "scenes": [
      {
        "emoji": "🚗",
        "tag": "Авария",
        "title": "Столкновение на перекрёстке",
        "text": "Вы попали в небольшое ДТП...",
        "choices": [
          { "icon": "📋", "text": "Оформить европротокол", "type": "good", "cost": 0 },
          { "icon": "🤝", "text": "Договориться на месте", "type": "bad", "cost": 50000 }
        ],
        "goodResult": { "title": "Верно!", "text": "...", "impact": "Сэкономили 50 000 ₽", "impactType": "save" },
        "badResult":  { "title": "Ошибка", "text": "...", "impact": "Потеряли 50 000 ₽",  "impactType": "loss" },
        "factoid": "Европротокол позволяет оформить ДТП без ГИБДД..."
      }
    ]
  },
  "demo": false,
  "warning": null
}
```

**Поле `warning`** (когда `demo: true`):

| Значение | Причина |
|----------|---------|
| `no_key` | Не задан `GIGACHAT_CLIENT_API` |
| `token_failed` | Не удалось получить токен |
| `insufficient_balance` | Недостаточно баланса на аккаунте |
| `network` | Сетевая ошибка |
| `api_error` | Ошибка GigaChat API |

## Запуск

### Требования

- Node.js 18+
- npm 8+

### 1. Установка зависимостей

Из корня проекта — установит пакеты и для бэкенда, и для фронтенда:

```bash
npm install
```

### 2. Установка доп. библиотек для API

Из корня проекта - установит пакеты бэкенда:

```bash
npm install uuid
```

### 3. Настройка окружения

```bash
cp backend/.env.example backend/.env
```

Откройте `backend/.env` и вставьте ключ GigaChat API:

```env
GIGACHAT_CLIENT_API=ваш_ключ_здесь
PORT=3000
```

> Без ключа приложение будет работать в **демо-режиме** с готовыми сценариями.

### 4. Запуск (разработка)

Откройте **два терминала**:

**Терминал 1 — бэкенд:**
```bash
cd backend && npm start
```
Сервер запустится на `http://localhost:3000`

**Терминал 2 — фронтенд (с hot-reload):**
```bash
cd frontend && npm run dev
```
Dev-сервер запустится на `http://localhost:5173`

Откройте браузер: **[http://localhost:5173](http://localhost:5173)**

---

### Продакшн-сборка

```bash
cd frontend && npm run build
```

После сборки бэкенд автоматически раздаёт фронтенд из `frontend/dist`.  
Достаточно запустить только бэкенд:

```bash
cd backend && npm start
```

Приложение будет доступно на **[http://localhost:3000](http://localhost:3000)**

## Переменные окружения

| Переменная | Обязательна | Описание |
|-----------|-------------|----------|
| `GIGACHAT_CLIENT_API` | Нет | Client Secret для GigaChat API (Сбер). Без неё работает демо-режим |
| `PORT` | Нет | Порт бэкенда (по умолчанию `3000`) |
